Why LogsGo?

A focused stack for push-based logs: fast write path, durable tiers, a purpose-built query language, and a dashboard you can run alongside the binary.

◆

gRPC Ingestion

Push logs with a lightweight Go client over gRPC — single entries or batches.

◆

Multi-Tier Storage

Memory (skiplist) → BadgerDB local → S3-compatible cloud, chained with transparent fallback.

◆

LogsGoQL

Custom query language with AND/OR, equality, and CONTAINS — instant and range queries.

◆

Web Dashboard

Built-in UI to query and visualize logs with tables and range charts.

◆

Auth & TLS

Optional JWT on REST and gRPC, plus TLS termination via config files.

◆

Queue Ingest

Optional RabbitMQ path for high-throughput async ingestion with worker pools.

◆

Compaction

Compact object-store blocks on a schedule for faster deep queries.

◆

Helm Chart

Ship to Kubernetes with the included Helm chart and configurable ingress.

Architecture at a glance

Logs land on gRPC, sit in an in-memory skiplist, then flush on time or size into BadgerDB and optionally S3-compatible storage. Queries walk the chain with deduplication.

Ingest

gRPC UploadLog / UploadLogs, or publish batches to RabbitMQ for async workers.

Tiered retention

Hot memory → warm local disk → cold object storage, each tier forwarding via .next.

Query

LogsGoQL plans execute concurrently across stores; REST powers the embedded UI.

Quick start

Run the interactive stack from examples/ or build from source.

# Clone and run the example stack
git clone https://github.com/Saumya40-codes/LogsGO.git
cd LogsGO/examples
docker compose up

# Dashboard: http://localhost:19091
# REST API:  http://localhost:8080
# gRPC:      localhost:50051
